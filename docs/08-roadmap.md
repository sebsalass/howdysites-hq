# 08 — Roadmap

## Phase 0 — Setup (Week 1)

- [ ] All three founders clone the repo, open it in Claude Code, read the docs
- [ ] Agree on lane assignments (`docs/03-roles.md`) and log it in `memory/decisions.md`
- [x] Agency name: **Howdy Sites** (howdysites.com — registering now); repo renamed to howdysites-hq
- [ ] Fable 5 builds dashboard steps 1–2 (kanban usable)
- [ ] Set up: Google Places API key, outreach mailbox on a secondary domain (start warming it — takes ~2 weeks), Stripe account
- [ ] Build OUR OWN site (we cannot sell websites without one) — it's also our portfolio piece #1

## Phase 1 — Proof (Weeks 2–6): first 5 clients

- [ ] First scrape run: 1 city, 2 niches (per `config/targets.json`)
- [ ] 100 leads in the pipeline, outreach running daily
- [ ] First mockup demos sent
- [ ] **Close clients #1–5** — even at a discount ("founding client" pricing is fine, log it)
- [ ] Deliver all 5 inside the 7-day promise; get testimonials + before/after screenshots
- [ ] Form the LLC + business bank account before invoice #1
- Exit criteria: 5 paying clients, real funnel numbers replacing the assumptions in `docs/02-business-model.md`

## Phase 2 — System (Months 2–4): make it repeatable

- [ ] Dashboard fully built (steps 3–6), weekly metrics reviewed every sync
- [ ] All 3 cities live, 4+ niches
- [ ] Pricing tuned from real close-rate data (update `pricing.json`, not vibes)
- [ ] Care-plan health reports going out monthly
- [ ] Target: **25–30 total clients, ~$2.5k+ MRR** from care plans
- Exit criteria: a week where the machine runs on cadence — scrape Monday, outreach daily, 1+ close, 1+ launch — without heroics

## Phase 3 — Scale (Months 5–12)

- [ ] 100 care clients / ~$10k MRR goal
- [ ] Launch expansion revenue (GBP optimization, review-gen, local SEO packages)
- [ ] Decide: hire a VA for outreach? productize (self-serve tier)? new metros (Austin, Fort Worth)?
- [ ] Revisit the even three-way split → formalize equity/comp in decisions.md

## Standing rules across all phases

- Numbers over narrative: every weekly sync starts with the dashboard, not opinions
- Any change to pricing/model = edit JSON + log decision + push, same day
- Churn gets a post-mortem: every cancelled care plan gets a one-line "why" in decisions.md
