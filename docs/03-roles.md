# 03 — Roles

Three founders, three lanes. Everyone can do everything in a pinch (the playbooks + Claude Code make each lane runnable by any of us), but each lane has one owner so nothing falls through.

> Lane assignments below are a **starting proposal** — swap names as you figure out who's actually good at what. The lanes themselves shouldn't change.

## Lane 1 — Pipeline (lead gen + data) — **Owner: Sebas**

- Runs the scraping/audit workflow weekly per `docs/04-lead-gen-playbook.md`
- Keeps `data/leads/` clean: fresh leads in, dead leads marked, no duplicates
- Owns the dashboard and this repo (merges, structure, tooling)
- Reports the weekly numbers: leads added, contact rate, reply rate

## Lane 2 — Sales (outreach + closing) — **Owner: Michael**

- Works the lead list daily: sends outreach per `docs/05-sales-playbook.md`, logs every touch in the lead's file
- Runs demo calls, presents the mockup, closes, collects payment
- Owns the outreach scripts — updates them in the playbook when something works or flops
- Hands closed deals to Delivery with the intake form filled in

## Lane 3 — Delivery (build + care) — **Owner: Parker**

- Builds sites per `docs/06-delivery-playbook.md` (with Claude Code doing the heavy lifting)
- Manages launches: domain, hosting, Google Business link, handoff walkthrough
- Runs the care plan: edit requests, uptime, the monthly client health report
- Owns client happiness after the sale — care-plan churn is Lane 3's number

## Shared rituals

- **Weekly sync (30 min):** each lane owner reports their number (leads added / deals closed / sites live + churn). Decisions made get written to `memory/decisions.md` before the call ends.
- **Everything through the repo:** if it isn't committed and pushed, it didn't happen. Each person `git pull`s at the start of any work session.
- **Money:** even three-way split of profit until we decide otherwise (log any change in decisions.md). All revenue into one business account once the LLC exists.
